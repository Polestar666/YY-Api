# 项目使用说明

## 1. 项目简介

这是一个本地运行的 AI API Gateway 管理项目，提供：

- 供应商管理
- 网关 API Key 管理
- 登录鉴权
- OpenAI 兼容接口转发
- 诊断与测试页面

## 2. 环境要求

- Node.js
- npm

## 3. 安装依赖

如果本地还没有依赖，先执行：

```bash
npm install
```

## 4. 启动项目

执行：

```bash
npm start
```

默认服务地址：

- `http://localhost:3001`

启动后会提供这些页面：

- 登录页：`http://localhost:3001/login.html`
- 供应商管理：`http://localhost:3001/api-manager-ui.html`
- 网关 Key 管理：`http://localhost:3001/api-server-ui.html`
- 诊断页：`http://localhost:3001/diagnose.html`
- 测试页：`http://localhost:3001/test-api.html`

## 5. 登录方式

默认管理员账号：

- 用户名：`admin`
- 密码：`admin123`

登录后可访问受保护的管理接口和页面。

## 6. 核心功能

### 6.1 供应商管理

可在供应商管理页面中：

- 查看供应商列表
- 新增供应商
- 编辑供应商
- 删除供应商
- 为供应商维护模型列表

### 6.2 网关 Key 管理

可在 Key 管理页面中：

- 查看当前有效的网关 API Key
- 生成新 Key
- 保存 Key
- 删除多余 Key

> 说明：系统初始 Key 已写入 `data/gateway.json`，可在页面中查看和维护。

### 6.3 接口转发

项目提供 OpenAI 兼容接口：

- `GET /v1/models`
- `POST /v1/chat/completions`
- `POST /v1/responses`
- `POST /v1/messages`

健康检查接口：

- `GET /v1/health`
- `GET /health`

## 7. 数据文件

项目数据保存在 `data/` 目录下：

- `data/gateway.json`：网关 Key 和默认用户
- `data/vendors.json`：供应商与模型配置

## 8. 常用操作流程

### 8.1 先登录

1. 打开登录页
2. 输入管理员账号密码
3. 登录成功后进入管理界面

### 8.2 添加供应商

1. 打开供应商管理页
2. 新增供应商
3. 填写 API 地址、API Key、供应商类型
4. 保存后同步模型列表

### 8.3 测试接口

1. 打开诊断页或测试页
2. 先检查健康状态
3. 选择模型
4. 发送测试请求

## 9. 注意事项

- 本项目默认监听 `3001` 端口
- 修改 `data/*.json` 后，页面刷新即可生效
- 生产环境请自行替换默认账号和 Key
